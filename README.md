Я использую кэширование вот здесь 
```python
@lru_cache(maxsize=128)
def get_cached_tasks(user_id: int, search: str, sort_by: str):
    pass
```
- Почему именно этот эндпоинт:
  1. GET /tasks выполняется чаще всего (основной запрос пользователей)
  2. Чтение происходит гораздо чаще записи/изменения задач
  3. Данные не требуют мгновенной актуализации (допустима задержка)
  4. Снижает нагрузку на базу данных при частых запросах
    
- Параметры кэша:
  - user_id: ID пользователя (изоляция данных)
  - search: поисковый запрос
  - sort_by: поле сортировки
    
maxsize=128: достаточно для большинства пользователей
